# Универсальная SaaS CRM для Ресторанов (Архитектура)

## Главные правила разработки:
1. **Архитектура Multi-Tenancy:** Код должен быть полностью универсальным. Никаких захардкоженных названий (типа "Сары Алтын"). Каждый ресторан — это отдельный `Tenant` со своим `restaurant_id`.
2. **База данных:** Все данные (пользователи, заказы, столы) жестко привязаны к `restaurant_id`. Один ресторан никогда не должен видеть заказы другого.
3. **Безопасность:** Вход в систему только по логину (email) и паролю из базы данных. Пароли должны сверяться (или хэшироваться).
4. **Стек:** Node.js + TypeScript (Бэкенд), React (Фронтенд).
